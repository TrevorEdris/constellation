# Constellation (Codex / generic)

This is the Codex bootstrap for the constellation skills plugin. It mirrors the intent of `CLAUDE.md` for tools that read `AGENTS.md`.

## Skills
Skills live in `skills/<name>/SKILL.md` and are discovered from `~/.agents/skills/constellation` after running `scripts/install.sh` (or the manual symlink in `.codex/INSTALL.md`). The full list is in `CATALOG.md`. Invoke a skill explicitly with `/skills` or `$mention`, or rely on implicit matching against each skill's `description`.

For Claude-Code tool names used inside skills (`Task`, `TodoWrite`, etc.), see `skills/_shared/platform/codex-tools.md`. Subagent-based skills require `[features] multi_agent = true` in `~/.codex/config.toml`.

## Standing rules (full text in `rules/`)
- Discover → Plan → Implement with an explicit approval gate; PLAN passes `plan-validator` (≥ 70) before being shown. Maintain `.ai/sessions/` docs automatically.
- TDD (RED-GREEN-REFACTOR) for behavioral code; verify by running, not by reasoning.
- Concise output; confidence levels; never use the section-sign character; personas only in live conversation, never in committed files.
- Git safety: branch check, never push to main without approval, never commit secrets.

## Not available on Codex
Hooks, output-styles, and parallel subagents (without `multi_agent`) are Claude-Code-only. Their intent is preserved as rules and skill instructions. See `docs/PARITY-MATRIX.md`.
